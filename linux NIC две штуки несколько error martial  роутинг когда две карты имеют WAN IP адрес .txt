так вот мы щас создадим две таблицы маршутизации. одна будет встурпуть в силу
для пакетов если у нас сорс ип 1.1.1.1
а вторая если сорс ип 2.2.2.2  где 1.1.1.1 это ip ens4 а 2.2.2.2 это ip ens5


 1.1.1.1  ens4
 2.2.2.2  ens5



делаем это так


# cat /etc//iproute2/rt_tables | grep -v '#'

100 T1   	<<<<<<<

далаее их нужно наполнить. чтобы эти суки посреперезагрузки не исчелли 
делаем это так






    # cat /etc/network/interfaces
source /etc/network/interfaces.d/*

auto lo
iface lo inet loopback




auto ens4
iface ens4 inet static
    address 1.1.1.1/24
    gateway 1.1.1.10
    

    up ip route add default via 1.1.1.10 dev ens4 table T1 onlink
    up ip rule add from 1.1.1.1 table T1
    down ip rule del from 1.1.1.1 table T1
    down ip route del default via 1.1.1.10 dev ens4 table T1 onlink




auto ens5
iface ens5 inet static
    address 172.16.0.5/24
    
    up ip route add default via 172.16.0.1 dev ens5 table T2 onlink
    up ip rule add from 172.16.0.5 table T2
    down ip rule del from 172.16.0.5 table T2
    down ip route del default via 172.16.0.1 dev ens5 table T2 onlink



самый цимес это вот эти две комкнды


    # ip route add default via 1.1.1.10 dev ens4 table T1 onlink
    # ip rule add from 1.1.1.1 table T1


первая комнада содает правло в таблице T1 ( сама таблица создана путем редактрования файла
cat /etc//iproute2/rt_tables)

для этого второе правло. оно сует эту таблицу в  ip rule  который ядром просматриватется 
как раз для того чтобы понять в какуб табилицу марщртизациии надо идти. прчием 
ядро подйтет в тблицу T1 толкьо для пакетов  у которы SRC_IP=1.1.1.1  тоесть для пакетов
который имеют src ip от карточки ens4 
вот именно таким макаром мы завстить линукс  срать в сеть пакеты которые влеелтели в ens4
