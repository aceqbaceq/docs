как защитить прокскс хост от l2 броадкаст шторма
делае это силами ebtables
доавляем  вконфиг сети например для vmbr0
в целом несильно важно куда его вствялять в конфиге сетевом
потому что защита раобтает для всех портов сразу


auto vmbr0
iface vmbr0 inet manual
        ovs_type OVSBridge
        ovs_ports bond0   vlan4044
        ovs_mtu 9000
*       post-up ebtables -F INPUT
*       post-up ebtables -A INPUT -d Broadcast --limit 300/sec --limit-burst 300 -j ACCEPT
*       post-up ebtables -A INPUT -d Broadcast --limit 5/minute --limit-burst 1 --log --log-prefix "BCAST_STORM_DROP: " --log-level info -j DROP
*       post-up ebtables -A INPUT -d Broadcast -j DROP

как посмтреть статистику в ebtables


# ebtables -L INPUT --Lc

Bridge chain: INPUT, entries: 3, policy: ACCEPT
-d Broadcast --limit 303/sec --limit-burst 300 -j ACCEPT , pcnt = 102 -- bcnt = 7860
-d Broadcast --limit 5/min --limit-burst 1 --log-level info --log-prefix "BCAST_STORM_DROP: " -j DROP , pcnt = 0 -- bcnt = 0
-d Broadcast -j DROP , pcnt = 0 -- bcnt = 0


pcnt - означает число пакетов котоырые попали в это правило
прикол в том что ключ --c он обнуляет сатистику
так что это число пакетов с предудыщуго момента срабатывавания этого правиа
