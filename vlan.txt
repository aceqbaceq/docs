на самомо деле:


пусть у нас сеть это несклоько комутаторов. на них порты. к портам
подключены сетвые устройства.
без влан устроуйство с любого порта может достучать до устройства на любом другом порту
все порты равнодоступны.

теперь мы делаем вот что. мы на кажом коммутаторре половину портов суем в vlan1
а вторую половину в vlan2

у нас в итоге каждый физ комутатор виртуально разрезается на два независимых вирт коммутатора.
причем если раньше каждый коммутаттор был связан с каким то другим комуаттором или комутаторами
то теперь эти два коммутаттора виртальных никак друг с другом не связаны. 
и у нас сетевая доступность одного порта  с другого порта будет точно такая как если бы
мы распилили все физ коммутаторы на две штуки. а потом создали два новых комутатора. 
в первый войдут порты со всех фи комутаторов входящие в vlan1 а во второй все порты из физ
комутаторов входящих в vlan2
и трафик между этими двумя комутаторами никак не может пронкнуть друг в друга.
пример есть три без влан комутатора 

     ==============               ==============             =============
    |  физ комм A  |             |  физ комм B   |          |  физ комм C |
     ==============               ===============            ==============
     |   |    |   |               |   |    |   |             |   |   |   |


       |____|___|___________________|____|___|_________________|___|____|
        

так вот устройтво подлюченное к любому порту любого комутатора
может достучаться до любго порта любого комутатора я это показал нижней зубчатой хренью



теперь теже комутаторы но на каждом у нас два вилана и еще есть транковый порт

     ==============               ==============             =============
    |  физ комм A  |             |  физ комм B   |          |  физ комм C |
     ==============               ===============            ==============
     | 1 | 2  | t |               | 1 | 2  | t |             | 1 | 2  | t |
              
                |_____________________________|__________________________|

1,2 это указано в каком вилане этот порт силит а также номер этого порта на комутаторе по счету.
 t - означает что этот порт транковый.
теперь у нас устройство пдлкюченное к порту любого комутатора и этот порт если влан1 может
длстучаться только до других портов которые тоже в влан1. и у нас эта схема полносью
эквилваентна схеме


     ================               ================         
    |  вирт комм 1   |             |  вирт комм 2  |        
     ================               ================        
     | A1 | B1 | С1 |               | A2 | B2  | C2 |            
       
        |___|_____|                    |____|_____|



A1, B1, C1 - указывает то что в этот комутатор входят порты с физ комутаторов A,B,C их порты 
с номером 1. 

и мы получаем два коммутатора устройста которые к ним подключены они друг с другом не 
могут достучаться. полная изоляция. !

транковые порты это порты через которые прогоняется трафик из обоиз виланов.
в данном случае они используются чтобы перегонять трафик с одного физ коммуттатора на другой
в рамках одного вилана
