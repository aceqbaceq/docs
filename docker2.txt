выяснилось что оба они в рутлес моде немогут прикосединить контейнрер
к существующему бриджу. может и могут тока уй знает как это сделать.

поэтому идем  пперекрючаемся в контест докер рут сервиса

 # docker context set default


так вот запускаем докер контейнер с пониженынми правами вот так

$ sudo docker run -it --name klava --network=bridge -u 1000:1000 alpine sh

ключевые ммоенты

-u 1000:1000   = это заупстить контейнер под обычным юзером
--network=bridge  = а это покючить к бриджу docker0


а вот эта хрень

 $ cat /etc/docker/daemon.json 
{
  "iptables": false,
  "ip-masq": false,
  "bip": "172.17.9.1/24"
}


она заставить докер не трогать иптейблс правила.
а эта штука "bip": "172.17.9.1/24"  она укажет какие ип адреса нужно назначать
