dns view 
это вот что.
авторитивный сервер без вью он на запрос от клиетета леезевт в один и тотже файл
и оттуда берет инфо о возваращет кленту.

а вью дает то что база одной и тожйже зоны теперь имеет две копиии нарпимер.
и если прилтает запрос от IP1 то такой клиент относится к вью1 и для него поиск по зоне
идет в файле1 а если запрс прилает от клиента2 то для него поиск для этой же зоны идет в файле2.

вот что дают view


вот пример от нейронки


view "internal" {
    match-clients { 192.168.0.0/24; }; // Только для локалки
    zone "example.com" {
        type master;
        file "internal/example.com.db"; // С локальными IP
    };
};

view "external" {
    match-clients { any; }; // Для всех остальных
    zone "example.com" {
        type master;
        file "external/example.com.db"; // С публичными IP
    };
};
