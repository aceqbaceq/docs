Usage: qaucli [options]


Legends:

   <hba instance>    - Instance number of an HBA port.
   <hba wwpn>        - World Wide Port Name of an HBA port.
                       (xx-xx-xx-xx-xx-xx-xx-xx or xxxxxxxxxxxxxx).
   <all>             - All adapters
   <target wwnn>     - World Wide Node Name of a target device.
                       (xx-xx-xx-xx-xx-xx-xx-xx or xxxxxxxxxxxxxx).
   <target wwpn>     - World Wide Port Name of a target device.
                       (xx-xx-xx-xx-xx-xx-xx-xx or xxxxxxxxxxxxxx).
   <target portid>   - Port ID of a target device.
                       (xx-xx-xx or xxxxxx).
   <target id>       - ID of a target device.
   <lun id>          - Logical Unit Number of a LUN.
   <lunname>         - The udev persistent device name to be assigned to
                       a designated LUN (Linux only).  

Options:

   [int]             - Interactive mode (Menu driven).

   -e [<hba instance>|<hba wwpn>all>] enable  
		             - Enabling the adapter legacy BIOS boot mode.
   -e <hba instance>|<hba wwpn> disable

		   For details, see Selectable Boot settings below:
		   Selectable  Boot Port Name     Device 
           Boot        /LUN               Boot
           ----------   --------------    -----------
           Disabled     X                 BIOS configures the first disk drive it
                                          finds as boot device.
           Enabled      None specified    BIOS configures the first disk drive it
		                                  finds that is also a LUN 0 as boot device.
           Enabled      Specified         BIOS scans through the specified WWPN/LUN
                                          until it finds a disk configured as the
                                          boot device.

   -e <hba instance>|<hba wwpn> disable all <boot mode>
                     - Disabling UEFI boot configuration. This command clears the boot 
		               device database entries and disables Adapter Driver, Selective 
		               Login, Selective LUN Login and FC NVMe parameters.
	       
		   ========================================================================
		   Boot Parameter      Alias Default  Description 
		   ========================================================================	
		   Adapter Driver      EB    Disabled  Enables or disables the UEFI driver.
	                                           To boot from a Fibre Channel disk
		                                       the UEFI driver must be enbled.                             
		   Selectivelogin      SL    Disabled  Login method that restrics device
		                                       logins to the adapter port to those
		                                       devices in the boot WWN database.
		                                       Enables to allow only device listed
		                                       can login.
		                                       Disables to allow any device to login.
		   Selectivelunlogin   LL    Disabled  Login method that restricts LUN logins
		                                       to the adapter port to those LUNs 
                         			            associated with a device in the boot
		                                       WWN database.
		                                       Disables to allow any LUN associated
		                                       with a device to login.
		   Worldlogin          WL    Disabled. Login method that enables or disables
		                                       forced world login. When enabled, this
		                                       parameter allows all devices to log in
		                                       to the adapter, overriding all other
		                                       login methods. Enabling World Login is
		                                       is typically done to troubleshoot FC
		                                       link and target device issues, which can
		                                       significant increase the boot time.      
		   FabricassignbootLun FA    Enabled   Fabric assign boot LUN discovery
		                                       allowsto eliminate the manual boot
		                                       LUN configuration of each adapter
		                                       from individual serves.
		   NVMeEnable          NE    Disabled  Enables or disables the FC NVMe storage
		                                       feature.
		                                       When enabled, the driver will search
		                                       for NVMe storage based on boot
		                                       database entries	
		   ========================================================================
		                 Table 2: UEFI Boot Settings

		      <target wwnn>      World Wide Node Name of a target device.                                 
		      <lun id>           Logical Unit Number (LUN) identifier number.

   -b <hba instance>|<hba wwpn>|<all> -op <update mode> [<-src> <template file>] 
      [--ports]
                     - Updates boot code/firmware across multiple HBAs
                       using a flash update configuration file.
       -op            - Specifies flash update mode type when using
                       template configuration file.
            0          - Force to update boot code/firmware across multiple
                        HBAs (Override mode).
            1          - Force to update boot code/firmware across mulitple
                       HBAs that are currently at or below a known revision.
                       HBAs that are currently up revision will be ignored.
            2          - Force to downgrade boot code/firmware across multiple
                       HBAs that are currently up revision.
                       Force to update boot code/firmware across multiple
                       HBAs are currently down revision.
                       HBAs that are currently at revision will be ignored.
       -src            - Use custom template file instead of the default
                       template file.
       <template file> 
	                 - User-defined configuration template file name.
                       See the sample template file included for a complete
                       syntax/format of the file.
                       If the <-src> option is omitted, the application will
                       try to use the default template file.
                       The default template file name is 'flashcfg.properties'.
					   
       --ports       - Optional option: Repeat flash update to all other ports.
